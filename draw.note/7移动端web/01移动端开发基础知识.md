+   屏幕
    ```
        1.移动端与PC设备最大的差异在于屏幕,这主要体现在屏幕尺寸和屏幕分辨率两个方面;

        2.通常我们所说的屏幕尺寸指的是屏幕的对角线长度(一般用英寸来度量),比如:一般我们所说的手机屏幕为5.5寸是指屏幕的对角线长度为5.5寸;

        3.分辨率,是指水平和垂直方向的像素数;例1920*1080指垂直方向有1920个像素,水平方向有1080个像素;
        
        - 注意: 按我们的习惯来说1920*1080,前者应该指水平方向,后者指垂直方向,而在移动端中恰恰相反;

        4.相对长度单位和绝对长度单位:
            
            绝对长度单位是指它的大小无论在什么地方都是指固定的大小;
            
            相对长度单位的大小会受不同的情况而变化;

        5.像素密度:DPI\PPI(Pixels Per Inch) 每一英寸中显示的像素数量称为像素密度
        
        6.如何计算像素密度:
            使用勾股定理: 5英寸的手机,分辨率为1920*1080
                        解: Math.sqrt( Math.pow(1920,2) + Math.pow(1080,2) )/5;
                        结果就是每英寸上的像素数量
        7.PPI越大,展现出的画面越精细
            - Retina即视网膜屏幕,PPI大于320的屏幕被称为视网膜屏幕;
            - 当屏幕尺寸固定的时候,PPI越大像素的实际大小就越小;
    ```
+   设备独立像素:
    ```
        - 在相同大小,分辨率不同的屏幕上;同一张图片在其上面显示的大小不同;
        - 为了使不同屏幕统一布局的大小,可以使用一个单位,在不同的屏幕上代表的像素大小不同,这个单位就叫做设备独立像素;
        - ios上称为PT(point),android叫DP或者DIP

    ```
+   物理像素和css像素
    ```
        物理像素是指当前屏幕中,一像素的大小;
        css像素是指网页布局时用到的单位,其默认值是和物理像素保持一致的;但是我们可以通过放大和缩小改变其大小;

    ```

+   viewport的设置
    ```
        通过meta来设置viewport
        < meta name="viewport" content="width=device-width">
        // 设置ideal-viewport 等于 layout-viewport

        
        // 设置ideal-viewport/layout-viewport ==1 效果和上面一样,为了兼容两者都要写

        <meta name="viewport" content="minimun-scale=1">
        minimun-scale :设置最小的缩放比例,默认显示最小
        maximum-scale: 设置最大的缩放比例
        user-scalable = yes/no 设置是否允许用户通过捏合手势缩放;
    ```
